# Roadmap de Desenvolvimento

## PARTE 1 — Sistema de Draft

### Feature 1 — Tela de Configuração
**Status:** Implementada.
Permite configurar número de participantes, jogadores por equipe, seleções disponíveis e regras do draft.

### Feature 2 — Sorteio Automático da Ordem
**Status:** Implementada.
Sorteia aleatoriamente a ordem dos participantes.

### Feature 3 — Draft de Seleções
**Status:** Implementada.
Cada participante escolhe sua seleção.

### Feature 4 — Draft de Jogadores
**Status:** Implementada.
Escolha dos jogadores com pool atualizável.

### Feature 5 — Consolidação dos Elencos
**Status:** Implementada.
Exibe todos os jogadores selecionados por cada equipe com IDs e posições.

### Feature 6 — Exportação dos Elencos
**Status:** Implementada.
Gera `draft-copa-2026.json` com jogadores, times e playerids do FC26.

### Feature 7 — Chaveamento
**Status:** Implementado.
Mata-mata de 16 avos até a final.

### Feature 8 — Registro de Resultados
**Status:** Implementado.
Inserir placares e avançar vencedores.

### Feature 9 — Persistência
**Status:** Implementada.
Salva automaticamente todo o estado do draft (configurações, seleção de países, picks, elencos) em localStorage a cada ação. Restaura automaticamente ao recarregar a página, mantendo a etapa exata onde o usuário parou.

## PARTE 2 — Conversor para RDBM 26

### Feature 10 — Conversor TXT (Python)
**Status:** Implementado (`converter_v2.py` + `converter.bat`).

### Feature 11 — Conversor Web (HTML)
**Status:** Implementado (`converter.html`).
Interface visual com tutorial passo a passo.

### Feature 12 — Database de Jogadores
**Status:** Implementada (`jogadores_final.json`).
1237 jogadores baseados na lista oficial de convocados da FIFA Copa 2026.

## PARTE 3 — Correção de IDs (PRIORIDADE MÁXIMA)

### Feature 13 — Validação e Correção dos PlayerIDs
**Status:** EM ANDAMENTO — PRIORIDADE MÁXIMA.

**Problema:** Dos 1237 jogadores na database, 892 (72%) têm playerid do FC26
identificado automaticamente. Os 345 restantes (28%) precisam de correção manual.

**Método:**
1. Usar FC Editor para buscar o playerid de cada jogador sem ID
2. Adicionar o ID no `jogadores_final.json` e no `MANUAL_PLAYER_IDS` do conversor
3. Validar cada ID conferindo nacionalidade + posição no FC26

**Jogadores pendentes (345):**
Lista completa disponível rodando `python validar_ids.py`

**Como ajudar:**
1. Abrir o Squad File no FC Editor by Decoruiz
2. Procurar o jogador pelo nome completo
3. Anotar o playerid (ex: 186555)
4. Informar para adicionar na database

### Feature 14 — Validação Automática de IDs
**Status:** Implementado (`validar_ids.py`).
Confronta cada playerid da database com os dados do FC26
(nacionalidade + posição + nome) para detectar IDs incorretos.
